glFusion evList Plugin Configuration

Main Options

Allow Anonymous View?
Select whether anonymous visitors can view the calendar. Note that permissions can be restricted for each event as well.
Who can add events
Select Users, Anonymous, or both as the user group that can create events.
Allow HTML in events?
Select whether HTML code can be added to event descriptions.
Show on user menu?
Select the link that will be added to the user menu, if any.
Enable Main Menu?
Select whether a link to EVList will be available from any navigation menu. Also adds the entry to the "Extras" menu.
Enable categories?
Select whether event categories will be used.
Enable comments?
Select whether to enable guest comments on events.
Enable Centerblock?
Select whether the upcoming event list will be shown in the centerblock.
Centerblock Position
Select the position of the centerblock, if used.
Centerblock Topic
Select which topic will display the centerblock.
Centerblock Range
Select the range of events that will be shown in the centerblock.
Events to show in list
Enter the maximum number of events to be shown in list views.
Events to show in blocks
Enter the maximum number of events to be shown in blocks.
Characters to show in summaries
Enter the maximum number of characters to show in event summaries.
Enable Reminders?
Select whether to allow visitors to enter their email address to receive a reminder of upcoming events.
When has an event passed?
Select the date/time when an event is considered to have passed for purposes of displaying upcoming events.
Default permissions
Select the permissions to be applied to new events by default. These may be overridden when the event is created or edited.
Reminder Speedlimit
Enter the number of seconds that must pass between reminder email address submissions.
Posting Speedlimit
Enter the number of seconds that must pass between event submissions.
Minimum Reminder Days
Enter the minimum number of days before an event that a reminder can be sent.
Display glFusion blocks?
Select which glFusion blocks will be displayed along with event listing or calendar views.
Default View
Select the view to be shown by default upon visiting the main EVList page.
Max days for Upcoming Events
Enter the maximum days in the future for events to be displayed in "upcoming events" blocks.
Use Locator Plugin?
If the Locator plugin is installed, it can be used to determine the locaiton of events, proviee a dropdown list of event locations, and display a map along with the event. Select whether this integration will be used.
If the Locator plugin is not installed this option has no effect.
Use Weather Plugin?
If the Weather plugin is installed it can display a weather forecast for the event location. If it is not installed this option has no effect.
Calendar Template
Select whether to use the original HTML calendars or the new JSON AJAX-enabled calendars. JSON is recommended unless you experience issues with it.
Enable Registrations
Select whether users are alowed to sign up for events. Tickets are created for each signup.
Enable Ticket Printing
Select whether users can print their own paid or unpaid event tickets. If you enable registrations but do not actually use tickets you can turn this off.
Ticket Format String
Enter a format string for creating ticket numbers. Evlist currently uses COM_makeSid() to create a unique number. See sprintf() for formatting options. This string must include "%s".
Meetup API Key
Enter your API key for meetup.com, if you wish to include events from Meetup in your calendars. See meetup.com for more information.
Meetup Group ID(s)
Enter one or more meetup.com group IDs to include in your calendars. Your API key must have permission to view the events in these groups (you must first join the groups). Separate multiple groups with commas, e.g. "12345,2468"
Cache Minutes
Enter the number of minutes to cache results from meetup.com. This is meant to avoid excessive requests to Meetup.
